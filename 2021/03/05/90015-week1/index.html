墨尔本大学 COMP90015 第一周知识点总结。

# 分布式系统的定义

分布式系统是指连接网络的许多计算机通过传递信息进行沟通以及指令的执行，也可以说是一群互相独立的计算机，它们沟通配合，所表现的功能对用户来说就像一个计算机一样。

# 分布式系统与网络的区别

  • 计算机网络更强调网络的组织，比如数据包和路由。
  • 分布式系统更强调系统的一致性和透明性，让用户感觉是在使用一台电脑。

# 分布式系统的成因

  • 功能的区分：不同的计算机可能容量不同或承担着不同的作用
    • 服务端和客户端
    • 数据的采集和处理
  • 固有的划分：
    • 不同的信息被创建
    • 需要支持工作上的合作
    • 对于工业上的系统化支持
  • 计算机性能上的分配不均
  • 可靠性：数据备份和保留
  • 经济成本：通过分享减少开销

# 分布式系统产生的结果

  • 并发：不同的任务自动进行，并且通过交换信息协调行动
  • 异质性（Heterogeneity）
  • 没有全局时钟（Global Clock）
  • 不会整个系统一起瘫痪

# 分布式系统的特性

  • 并行的任务：任务可以同时进行
  • 通过信息传递交流：信息没有共有的存储，而是通过传递交流
  • 资源分享：让资源被许多用户获得
  • 没有全局状态
  • 没有全局时钟

# 分布式系统的目的

  • 连接用户和资源
  • 透明性
  • 开放性
  • 可拓展性
  • 可获取性

# 分布式系统面临的挑战（重点）

# 异质性（Heterogeneity）

定义：各式各样的组成部分必须能交互操作

  • 用户使用不同的设备，比如 PC、平板、手机
  • 用户使用不同的操作系统
  • 用户使用不同的数据表现方式

# 分配的透明度（Distribution Transparency）

定义：分布对用户尽可能隐藏

  • 你邻居去加勒比海当海盗去了（你不应该知道的）

# 并发（Concurrency）

定义：对资源的共同获取

  • 不同的用户同时使用服务

# 容错（Fault Tolerance）

定义：系统的部分瘫痪不会导致整体瘫痪

  • 数据库崩了怎么办
  • 用户在使用过程中电脑崩了怎么办

# 安全性（Security）

定义：系统必须被正确使用

  • 有人使坏来偷你数据怎么办
  • 信息泄漏问题

# 可拓展性（Scalability）

定义：当用户增长时，系统应该能扩容，性能可以提升

  • 用户访问量过多

# 重复使用和开放性（Reuse and Openness）

定义：接口应该开放，便于更新

  • 你的软件在其他计算机运行
  • 技术的更新
