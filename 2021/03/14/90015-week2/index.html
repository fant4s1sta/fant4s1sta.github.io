墨尔本大学 COMP90015 第二周知识点总结。

# 端口（Port）

  • TCP 和 UDP 协议都是使用端口来进行不同应用对接收数据的匹配处理。
  • 端口用一个 16bit 的正整数表示：
    • ftp：21/tcp
    • telnet：23/tcp
    • smtp：25/tcp
    • http：80/tcp
    • login：513/tcp
  • 用户层面通常使用的端口都是大于 1024 的。

# 套接字（Socket）

  • 套接字提供了运输层里网络编程的接口。
  • 使用套接字的网络通讯与文件 I/O 非常相似。
  • 套接字通讯不需要相同的编程语言。

# 套接字通讯

  1. 我们需要一个 Server 在一台计算机运行并且拥有一个绑定到特殊端口的套接字。服务器需要等待和听取 Client 连接请求的套接字。
  2. 如果一切正常，Server 接收连接请求。同时，因为端口被用于连接，Server 会需要一个绑定新端口的新套接字，通过它可以在连接 Client 时听取原来套接字的连接请求。

# Java 套接字

Java 的.net 包中提供了两个类：

  • Socket：用于实现 Client
  • ServerSocket：用于实现 Server

  1. 创建 I/O 流来与 Client 通讯
  1. 进行与 Client 的通讯
  1. 关闭套接字
