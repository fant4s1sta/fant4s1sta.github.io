# 数据库系统的性能

数据库系统的性能由三个方面决定：

  • 硬件（Hardware）
    • 处理区速度
    • 处理区个数
    • 磁盘和 I/O 带宽
    • 主存大小
    • 网络
    • 架构类型
  • 软件（Software）
    • 所使用的数据库技术
  • 数据库优化（Database Tuning），故障恢复（Crash Recovery）
    • 索引
    • 重复数据
    • 共享数据

# 硬件

# 硬盘的构成原理

如图，这是一个快速旋转的圆盘（Platter），由一圈一圈的圆形轨道组成，我们叫它磁道（Track）。我们可以看到蓝色的扇形区域，这是磁盘扇区（Disk Sector），它也将我们的磁道分割出一小部分扇区。而连续的磁道扇区，能组成一个团（Cluster），也就是绿色的部分。另外，还有一个常常的执行器臂（Actuator Arm）用来读写我们的数据，如果我们想读取某个磁道，我们需要把执行器臂移动到那个磁道。

# 磁盘访问时间

访问时间 = 寻道时间（移动到磁道的时间）+ 转动时间（转动读写数据的时间）+ 数据传输时间（长度 / 带宽）

# SSD（固态硬盘）特点

  • 没有机械硬盘的移动部分
  • 使用的是硅而不是磁
  • 没有寻道 / 转动的延迟
  • 没有机械硬盘的启动时间
  • 运行时很安静
  • 访问快（100 微秒相较于机械硬盘的 2000-3000 微秒）
  • 贵！！！

# 命中率

命中数 = 缓存命中的 Reference / 所有的 RAM Reference
有效防蚊时间 = 命中率 * 缓存访问时间 + （1 - 命中率）* 存储访问时间

# 数据库系统类型

数据是这样被储存的：

  • 单个文件：每行一条记录，逗号或者分号分割
  • RDBS：由很多行很多列组成的表格，有主键来区分
  • 面向对象：数据储存在对象格式里（面向对象编程）
  • No-SQL：非关系型的数据可

# 数据库架构

  • 集中（Centralized）：数据集中在一起
  • 分布（Distributed）：数据分布在许多点
  • WWW：数据存放在全世界并且有许多拥有人
  • 网格（Grid）：类似分布式但是每一个点管理者自己的资源
  • P2P：类似网格但是每一个点可以随意加入或离开
  • 云：网格的泛化但是资源可以通过要求获得
